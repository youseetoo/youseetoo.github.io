OASIS Flow-Stop Microscope Software Documentation

This document provides a detailed guide to the software controlling the OASIS flow-stop microscope, based on ImSwitch. It includes setup instructions, interface descriptions, and operational details to ensure the correct functioning of the microscope for sampling and imaging purposes.

System Requirements​

  • Raspberry Pi: The system is powered by a Raspberry Pi with a custom operating system designed to run ImSwitch and control the microscope functions.
  • Power Supply: 12V at >4A (~50-60W) to ensure proper operation.
  • Data Storage: MicroSD card or an external USB storage device with sufficient space for saving images and videos.
  • Display: A connected monitor with a keyboard and mouse for direct control or through a remote desktop.
  • HIK Camera: The system uses a HIK camera for image capture and video recording.

Setup Instructions​

Step 1: Preparing the Hardware​

  • Ensure the microSD card is properly plugged into the Raspberry Pi.
  • Check that the silicon tubes for the flow-through system are immersed in the sample, and the tubes are free of particles or blockages.
  • Plug in the appropriate data storage device, either an external USB hard drive or an SD card.

Step 2: Powering the System​

  • Verify that the power supply is properly connected.
  • Use a 12V power supply with >4A to power the system.
  • Wait for the Raspberry Pi to boot up, which may take 1-2 minutes.

Step 3: Opening the OASIS Camera​

  • Unlock the camera box using the paddles and verify if the screen shows the desktop.
  • The screen saver will display a desktop background with icons for software actions:
    • Update Docker image for ImSwitch.
    • Launch ImSwitch in debugging mode.
    • Start ImSwitch in default OASIS mode (automatic on boot in the best-case scenario).

Step 4: Internet Connection (Optional)​

  • Connect to the internet by selecting any available SSID from the taskbar.
  • Establish a connection to enable remote desktop access via Raspberry Pi Connect for image streaming and debugging.

Connecting to the Internet​

If needed, follow these steps to connect the system to the internet:

  1. Scan for available Wi-Fi networks from the taskbar.
  2. Choose the appropriate SSID and connect.
  3. With a working connection, you can use the Raspberry Pi Connect for remote desktop control, image streaming, and debugging.

Launching the ImSwitch Interface​

Step 1: Automatic Launch​

  • On boot, the Chromium browser will open automatically in full-screen mode and load the ImSwitch interface from https://localhost:8001/imswitch/index.html.

Step 2: Manual Launch​

If the interface doesn’t open automatically:

  1. Open the Chromium browser from the desktop.
  2. Click the "ImSwitch" shortcut on the desktop.
  3. Run the launch_imswitch.sh script and select "Execute in terminal."
  4. Toggle full-screen mode with F11 or FN+F11.

Once launched, the interface will show live streaming options, motor controls, and the plugin section for image capture.

Data Collection Process​

Step 1: Starting the Stream​

  • Click on the "Start Stream" button to view live images of the sampled water.

Step 2: Starting the Plugin​

  • Click on the “Plugins” button to begin the data collection process.

Step 3: Configuring Sampling Parameters​

  • Enter metadata such as experiment name, description, volume per image, etc.
  • For continuous sampling, set the image number to <0 for infinite measurements.
  • The system captures images every few seconds and videos at 5 fps, saving them in MP4 format. To avoid memory corruption, the system creates new files every 2000 frames.

Motor Controls​

  • Focus (Z-axis): Control the focus using the + and - buttons. Adjust step sizes and speeds for fine-tuning.
  • Pump (X-axis): Similar controls are provided for the pump system.
  • Y-axis: Not used in this configuration.
  • The system remembers its current motor positions, and absolute coordinates can be set for precise movements.

Image and Video Recording​

  • Start/Stop Stream: Start or stop the live camera stream using the dedicated button.
  • Photo Capture: Use the camera icon to take a snapshot, saving it directly to the external storage.
  • Video Recording: Use the round record button to start/stop video recording, saving MP4 files to the external storage.

Metadata Entry​

  • Enter the metadata for the experiment, including the name, date, description, and sample volume per image.
  • Configure the image capture settings, specifying the number of images to take. The system can be set to capture indefinitely with images saved in JPEG format.

Troubleshooting​

  1. The interface doesn't load automatically:

    • Manually start Chromium and ImSwitch by following the steps in the Manual Launch section.

  2. No live stream appears:

    • Ensure the camera is correctly connected and powered.
    • Verify the stream settings and ensure the "Start Stream" button has been pressed.

  3. Motors not moving:

    • Check the power supply to ensure it provides sufficient voltage and current.
    • Verify the motor control settings in the ImSwitch interface.
