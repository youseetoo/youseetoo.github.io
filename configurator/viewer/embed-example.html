OpenUC2 OptiKit Embedded Viewer

Interactive 2D/3D viewer for OptiKit cube arrangements. Embed it in any documentation site, wiki, or web page.

1. Basic iframe Embed

Point the ?json= parameter to any hosted OptiKit layout JSON file:

<iframe
  src="https://youseetoo.github.io/configurator/viewer/?json=https://example.com/my-layout.json"
  width="100%"
  height="500"
  frameborder="0"
  style="border-radius: 8px; border: 1px solid #ddd;"
></iframe>

2. With Custom Polling Interval

Set ?interval= (in ms) to control how often the viewer re-fetches the JSON. Default is 5000ms.

<iframe
  src="https://youseetoo.github.io/configurator/viewer/?json=URL&interval=2000"
  width="100%"
  height="500"
  frameborder="0"
></iframe>

3. Inline Data (no server needed)

Base64-encode the JSON and pass it via ?data=. Useful for static docs or when you don't want to host a JSON file.

// In JavaScript:
const layout = { uc2_components: [...] };
const encoded = btoa(JSON.stringify(layout));
const src = `https://youseetoo.github.io/configurator/viewer/?data=${encoded}`;
iframe.src = src;

4. Dynamic Updates via postMessage

Push new layouts to the viewer at runtime without reloading. Useful for apps that generate/modify layouts dynamically.

// Send a layout object directly:
const iframe = document.getElementById('optikit-viewer');
iframe.contentWindow.postMessage({
  type: 'optikit-load',
  json: { uc2_components: [...], name: 'My Setup' }
}, '*');

// Or tell it to poll a new URL:
iframe.contentWindow.postMessage({
  type: 'optikit-load',
  url: 'https://example.com/updated-layout.json'
}, '*');

URL Parameters Reference

ParamTypeDescription
jsonURLURL to an OptiKit layout JSON file. Viewer polls this for changes.
dataBase64Base64-encoded JSON layout (static, no polling).
intervalNumberPolling interval in ms (default: 5000, min: 1000).

JSON Format

The viewer reads the standard OptiKit export format. Minimum required structure:

{
  "name": "My Setup",
  "uc2_components": [
    {
      "moduleId": "mirror-1x1",
      "originalName": "Mirror",
      "grid_pos": [3, 4, 0],
      "rotation": [0, 90, 0],
      "description": "45° beam steering mirror"
    }
  ]
}

Each component needs moduleId, grid_pos [x, y, layer], and rotation [rx, ry, rz]. The ry value (Y-axis rotation) determines the facing direction in the viewer.
